Unified Automatic Memory Pipeline

UAMP

How Claude Code's memory system evolved from 4-agent pipelines to zero-config persistence

Section 01

The Problem

Claude Code sessions are ephemeral. When a session ends, everything learned — decisions, context, relationships — vanishes. The next session starts cold.

For someone running infrastructure across multiple PCs, managing 4 nonprofit boards, operating a NAS with Docker, and maintaining a personal knowledge vault, this is unacceptable. Every session restart means re-explaining the same architecture, the same conventions, the same project state.

The question: how do you make Claude remember everything, automatically, across sessions and machines?

Section 02

UAMP v1 — The Full Pipeline

March 2026. The original design was ambitious: 4 specialized agents running automatically, capturing every tool call and distilling it into persistent knowledge.

V1 — MARCH 2026

The 4-Agent Architecture

  • 01
    claude-mem plugin — PostToolUse hook. Runs Haiku on every tool call to classify observations. Writes to SQLite.
  • 02
    Export agent — Stop hook. Reads claude-mem SQLite, produces Obsidian-formatted markdown in the vault. 315 lines of JS.
  • 03
    obsidian-git — 10-minute auto-commit cycle. Syncs vault changes from all sources (Code, Desktop, manual edits).
  • 04
    Nightly scheduled task — 22:00 run. Opus-powered knowledge extraction, auto-memory updates, SPEC changelog generation.
Tool call ──▸ claude-mem classifies (Haiku) ──▸ Export to Obsidian ──▸ Git sync ──▸ Nightly extraction (Opus) │ │ │ │ │ PostToolUse hook │ Stop hook (315 lines) │ obsidian-git │ Scheduled task │ ~739 calls/day │ SQLite → Markdown │ 10-min cycle │ 4 nightly jobs ▼ ▼ ▼ ▼ SQLite DB vault/sessions/ GitHub vault/knowledge/
$17 Monthly cost
~739 Haiku calls per day
35-40% Observation overlap
360 Lines of hook code

It worked. But a 4-agent expert review found the fundamental problem: 35-40% of observations overlapped with what native auto-memory already captured for free. Sessions were bloated with meta-operations. The pipeline was, in the reviewers' words, "over-engineered for the problem it solves."

Section 03

UAMP v2 — The Simplification

April 2, 2026. Radical simplification. Rip out everything except what Claude Code provides natively.

V2 — APRIL 2026

The Purge

Component V1 V2
claude-mem PostToolUse Active (all tool calls) Disabled entirely
Export agent 315-line Stop hook Removed
Nightly task 4 Opus-powered jobs Removed
Session persistence SQLite + Obsidian export Native auto-memory only
Monthly cost $17 $0
Hook code 360 lines 45 lines
Fragility 6/10 2/10

Clean. Zero cost. Two tiny hooks (vault-pull + session-end) totaling 45 lines. But something was lost.

Each session started with only CLAUDE.md context. No awareness of what happened yesterday. No nuanced decisions preserved. No relationship context. The 80% that auto-memory covered was fine. The 20% it missed — the why behind decisions, the cross-project connections — was gone.

$0 Monthly cost
45 Lines of hook code
0% Cross-session continuity
Section 04

UAMP v3 — The Unified Approach

April 11, 2026. The current architecture. Re-enabled claude-mem, but with a narrow focus: capture only meaningful work (Edit, Write, Bash, MCP calls), skip navigation noise (Read, Grep, Glob). Four persistence layers, three independent context sources at session start.

Native Auto-Memory

Facts per project, symlinked to vault/memory/. Auto-loaded every session. Zero cost. Covers feedback, project state, references.

Section 05

The Session Workflow

The daily rhythm that replaced the automated pipeline. Two independent skills, deliberately unbundled.

Work edit / build / research
→
/reflect SPEC.md + knowledge
→
/ship git commit + push

/reflect proposes SPEC.md changelog entries, detects knowledge worth capturing as vault entries, and checks if CLAUDE.md conventions need updating. It is the intelligence layer.

/ship commits and pushes to both ClaudeHub and vault repos. It is pure mechanics.

Key design decision: these are separate skills, not bundled. UAMP v1 failed precisely because bundling created coupled failure modes. When the export hook broke, git sync broke too. Independent skills = independent failure.

DESIGN PRINCIPLE

Decouple Intelligence from Mechanics

/reflect can fail without breaking /ship. /ship can fail without losing /reflect's analysis. The vault still auto-commits via obsidian-git as a fallback. Each component operates independently. No cascading failures.

Section 06

What Actually Works

Six lessons from three architectures, extracted through expert reviews and daily use.

//

Native auto-memory covers 80% of cross-session needs at zero cost. Don't build what the platform gives you for free.

>_

SPEC.md changelog IS the session handoff. The next session reads it first. No separate handoff mechanism needed.

db

claude-mem's value is the 20% that auto-memory misses: nuanced decisions, relationship context, things that aren't in any file.

^^

Skills read their own feedback. Each skill has a feedback.md. The system improves itself across sessions.

[]

Vault knowledge entries are the permanent record. Everything else is working cache. Design for this hierarchy.

<>

Split /reflect from /ship. Independent failure modes. Never bundle intelligence with mechanics.
