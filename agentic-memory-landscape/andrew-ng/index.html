Agentic Memory Landscape

Andrew Ng

Four agentic design patterns + memory as enabler

01 The Four Agentic Design Patterns

In a five-part blog series published March-April 2024, Andrew Ng distilled the emerging landscape of agentic AI into four design patterns. Unlike Weng's taxonomy (which maps cognitive structures), Ng's framework is operational -- it tells practitioners what to build. Each pattern is a distinct architectural choice that can be composed with the others. Memory is not a pattern itself but the substrate that makes the patterns effective across interactions.

Pattern 1
Reflection
The agent critiques its own output and iterates toward quality

Pattern 4

Enabler
Memory
Persistent state that makes all four patterns effective across sessions

02 Pattern Details

Reflection

Self-Improvement

The agent generates output, then evaluates it against criteria -- either self-generated or externally defined. The generator-critic loop runs iteratively: generate, critique, revise, re-critique, until quality thresholds are met or iteration limits are reached. This is the simplest pattern to implement and yields surprisingly large gains.

Generator → Output → Critic → Feedback → Generator (iterate)

Ng cites a striking result from coding benchmarks: a GPT-3.5-based agent with reflection achieved dramatically higher accuracy than the same model without reflection. The pattern works because the critic role catches errors that the generator role systematically makes, and each revision narrows the gap. The key insight is that iteration is often cheaper than capability -- a weaker model looping can outperform a stronger model in a single pass.

HumanEval (SQL): 87% → 95% with self-reflection

Tool Use

External Capability

The agent generates structured requests to external tools -- web search, code execution, API calls, database queries, calculators. Tool use extends the agent beyond what the LLM can do from its training data alone. Ng frames this as the pattern that bridges the gap between knowing and doing.

Agent → Structured Tool Call (JSON) → Tool Execution → Result → Agent

The pattern requires careful design of tool descriptions and schemas. The LLM must understand when to use a tool, which tool to choose, and how to format the request. Tool use is the most widely adopted pattern -- it is standard in ChatGPT, Claude, Gemini, and virtually every production agent framework. Ng notes that the quality of tool descriptions matters as much as the quality of the model's reasoning about them.

Planning

Autonomous Decomposition

The agent autonomously decomposes a complex goal into a sequence of steps, executes them, and adapts the plan based on intermediate results. Unlike simple chain-of-thought prompting, planning involves maintaining a persistent plan structure, tracking completion status, and re-planning when steps fail or new information emerges.

Goal → Decompose → Step 1 → Step 2 → ... → Re-plan if needed → Result

Ng identifies planning as the pattern with the most room for improvement. Current LLMs are good at decomposition but struggle with plan monitoring -- knowing when a step has failed and the plan needs revision. The pattern becomes essential for tasks that take minutes to hours, where the agent cannot rely on a single context window to hold the full problem state. Planning is what separates a chatbot from an agent.

Multi-Agent Collaboration

Specialized Roles

Multiple LLM instances are assigned specialized roles and collaborate through structured communication. One agent might be a coder, another a tester, a third a project manager. Each brings a different system prompt, tool set, and perspective. The collaboration can be hierarchical (manager delegates to workers) or peer-to-peer (agents debate and negotiate).

Orchestrator → Agent A (coder) + Agent B (reviewer) + Agent C (tester) → Synthesis

Ng highlights ChatDev and MetaGPT as exemplars -- systems where a software company's org chart is implemented as a multi-agent system. The pattern works because role specialization improves focus: an LLM prompted as "a security reviewer" catches vulnerabilities that the same LLM prompted as "a coder" would miss. The cost is coordination overhead and the need for well-defined communication protocols between agents.

03 The Benchmark That Changed the Conversation

Ng's most cited data point comes from the HumanEval coding benchmark. It crystallized the argument that agentic workflows on weaker models can beat stronger models used traditionally. This single comparison became the rallying cry for the agentic AI movement and justified significant investment in agent infrastructure.

HumanEval -- 164 Python programming challenges (Chen et al. 2021)

The 28-point gap demonstrates that agentic architecture can compensate for model capability. GPT-3.5 with reflection, tool use, and iterative refinement surpassed GPT-4 making a single attempt. This is not just an incremental improvement -- it suggests that how you use a model matters more than which model you use, at least for tasks where iteration is possible. The caveat: agentic workflows consume more tokens, take more time, and introduce failure modes that zero-shot does not have. The tradeoff is latency and cost for accuracy and reliability.

Ng uses this result to argue that even when a frontier model exists, it may be more economical to wrap a smaller model in agentic scaffolding. This has profound implications for deployment: organizations can achieve frontier-level performance on specific tasks using cheaper, faster models augmented with the right patterns.

04 Memory as the Fifth Dimension

Ng deliberately excluded memory from the four patterns. It is not a fifth pattern but a cross-cutting enabler -- the substrate that gives the other four patterns continuity across interactions. Without memory, reflection only works within a single session, tool results are lost, plans cannot span conversations, and multi-agent teams forget their coordination history. Memory transforms agents from stateless responders into persistent collaborators.

"Memory is not just another feature -- it's what transforms an AI from a tool you use to a colleague you work with. The four agentic patterns become dramatically more powerful when the agent remembers what worked, what failed, and what you prefer."

-- Andrew Ng, Microsoft BUILD 2024 keynote

Why Not a Pattern?

Patterns are architectural choices -- you can build an agent that uses reflection but not multi-agent, or tool use but not planning. Memory is different: it amplifies every pattern but is not a standalone capability. An agent with only memory and no patterns would just be a database with an LLM frontend.

MemGPT / Letta Connection

Ng partnered with Charles Packer (MemGPT creator) to build a DeepLearning.AI course on agent memory. MemGPT's approach -- treating context window management as a virtual memory system with hierarchical paging -- aligns with Ng's view that memory is infrastructure, not application logic.

Memory Enables Personalization

The BUILD 2024 talk emphasized that memory is what allows agents to become personalized. An agent that remembers your coding style, your preferred frameworks, your past mistakes -- that agent gets better over time. Without memory, every session starts from scratch.

Cross-Session State

Ng identifies the "fresh session problem" as the key blocker for production agents. Users build rapport and context over a conversation, then lose it when the session ends. Memory solves this by persisting relevant context across session boundaries, making the agent feel continuous rather than episodic.

The DeepLearning.AI course on memory (launched late 2024) covers practical implementation: how to decide what to store, retrieval strategies, memory compression, and the tradeoffs between different persistence backends. It operationalizes the BUILD 2024 vision into buildable components.

05 Key Facts

  • • Publication format: Five-part blog series on DeepLearning.AI newsletter ("The Batch"), published March-April 2024
  • • Series structure: Part 1: overview. Parts 2-5: one pattern each (Reflection, Tool Use, Planning, Multi-Agent). Memory discussed throughout as enabler
  • • Keynote: Microsoft BUILD 2024 -- expanded the framework with emphasis on memory as the critical missing piece in current agent deployments
  • • DeepLearning.AI courses: Multiple courses implement the patterns: "AI Agentic Design Patterns with AutoGen", "Building Agentic RAG with LlamaIndex", and the MemGPT/Letta memory course
  • • Influence: The four-pattern framework became the standard vocabulary for agent architecture discussions. "Agentic" as an adjective entered mainstream AI discourse largely through Ng's advocacy
  • • Key claim: Agentic workflows on GPT-3.5 outperform GPT-4 zero-shot on HumanEval (95.1% vs 67.0%). Most-cited datapoint in the agentic AI discourse of 2024
  • • Relationship to Weng: Complementary, not competing. Weng provides the cognitive science mapping (what types of memory exist). Ng provides the engineering framework (what patterns to implement). Together they form the descriptive and prescriptive layers of agent architecture
  • • Affiliation: Founder of DeepLearning.AI, co-founder of Coursera, former head of Google Brain and Baidu AI Group. The blog series reflects his educator role rather than research output
