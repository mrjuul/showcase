LangGraph

Two-layer memory architecture for stateful AI agents

Architecture

Checkpointers + Stores

LangGraph separates memory into two systems, both injected at compile time. Checkpointers handle thread-scoped conversation state; Stores handle cross-thread long-term memory with optional semantic search.

Checkpointers

Thread-scoped (short-term)
  • What: Saves full graph state at every superstep
  • Keyed by: thread_id + checkpoint_id
  • Enables: Resume, time-travel, human-in-the-loop
  • Backends: SQLite, Postgres, Redis, MongoDB, CosmosDB
+

BaseStore

Cross-thread (long-term)
  • What: Key-value store with namespace tuples
  • Keyed by: (namespace_tuple, key)
  • Enables: User profiles, preferences, semantic search
  • Backends: InMemory, Postgres, Redis

graph = builder.compile(checkpointer=cp, store=store)

Patterns

Key Patterns

New in 1.x

Runtime Context API

Runtime[Context] dataclass replaces the verbose config["configurable"] pattern. Provides typed access to user_id, store, and config. Requires defining a context_schema at graph definition.

Core Pattern

Dual Persistence

Checkpointer + Store compiled together. Short-term thread state and long-term cross-thread knowledge coexist with different persistence strategies and backends.

Higher-Level SDK

LangMem Three Types

Semantic (facts via create_memory_manager), procedural (prompt optimization), episodic (few-shot examples). Plugs into BaseStore for persistence. Works with create_react_agent.
