The canonical taxonomy for agent memory

01 Memory Taxonomy

Weng's blog post "LLM Powered Autonomous Agents" (June 2023) established the definitive mapping between human cognitive memory systems and their LLM equivalents. The taxonomy draws from cognitive psychology's standard model and shows how each memory type finds a natural analog in the architecture of language model agents.

Long-Term Memory Breakdown

Explicit (Declarative)

Episodic Memory

Personal experiences and events. LLM equivalent: logged past interactions, conversation history, experience replay buffers.

Semantic Memory

General world knowledge and facts. LLM equivalent: knowledge base, vector store of documents, structured databases.

Implicit (Procedural)

Procedural Memory

Skills and habits performed automatically. LLM equivalent: model weights from pretraining, fine-tuned behaviors, in-context learning patterns.

Implicit memory is encoded in the model's weights and cannot be easily inspected or modified at runtime, unlike explicit memory which resides in external retrievable stores.

02 Maximum Inner Product Search (MIPS)

Weng dedicates significant attention to the retrieval infrastructure that makes external memory practical. She frames the core challenge as Maximum Inner Product Search (MIPS) -- given a query embedding, find the stored vectors with the highest dot-product similarity. Exact search is O(n) and impractical at scale, so approximate algorithms trade accuracy for speed. These are the building blocks behind every vector database used in RAG and agent memory systems.

Algorithm Approach Key Property
LSH Locality-Sensitive Hashing. Maps similar vectors to the same hash bucket using random projection families. Query time is sublinear in dataset size. Probabilistic guarantees; tunable accuracy vs. speed via number of hash tables and bucket width. Works well in high dimensions but needs many tables for high recall.
ANNOY Approximate Nearest Neighbors Oh Yeah. Builds a forest of random projection trees. Each tree recursively partitions the space with random hyperplanes. Static index (build once, query many). Memory-mapped files allow sharing across processes. Developed by Spotify for music recommendation. Good for read-heavy workloads.
HNSW Hierarchical Navigable Small World graphs. Builds a multi-layer graph where each layer has exponentially fewer nodes. Search starts at the top and refines downward. Best-in-class recall at high throughput. Greedy traversal with logarithmic complexity. The default choice in most modern vector databases (Pinecone, Weaviate, Qdrant).
FAISS Facebook AI Similarity Search. A library (not a single algorithm) combining IVF (inverted file index), PQ (product quantization), and HNSW. GPU-accelerated. Industrial-grade; handles billion-scale datasets. Composable index types let you trade memory, speed, and accuracy. The most widely deployed ANN library in production.
ScaNN Scalable Nearest Neighbors by Google. Uses anisotropic vector quantization -- accounts for the direction of quantization error relative to the query, not just magnitude. State-of-the-art accuracy-speed tradeoff at the time of publication. Outperforms HNSW and FAISS on standard benchmarks (ANN-Benchmarks). Optimized for inner product search specifically.

Weng emphasizes that the choice of MIPS algorithm is not academic -- it directly determines the latency and quality of agent memory retrieval. A poorly-chosen index can make the difference between an agent that recalls relevant context in 10ms and one that either times out or retrieves irrelevant memories.

03 Generative Agents Case Study

Generative Agents: Interactive Simulacra of Human Behavior

Park et al. 2023

The Generative Agents paper is Weng's primary case study for how memory enables emergent behavior. Stanford researchers created a Sims-like sandbox with 25 LLM-powered agents who autonomously formed relationships, organized parties, and spread information through social networks -- all driven by a carefully designed memory architecture.

The Memory Stream

Each agent maintains a memory stream -- an append-only log of natural language observations. Every perceived event ("Klaus is painting at the studio"), conversation snippet, and internal reflection gets timestamped and stored. This creates a comprehensive record of the agent's subjective experience, forming the raw material for all downstream retrieval. The stream grows indefinitely; retrieval scoring determines what surfaces when the agent needs to act.

Three-Factor Retrieval Scoring

Recency
exp(-α × Δt)
Exponential decay based on time since last access. Recent memories surface more readily, mirroring the human recency bias. Decay rate α controls how fast old memories fade.
Importance
LLM score [1-10]
An LLM rates each memory's inherent significance. "Brushing teeth" scores low; "breakup with partner" scores high. Assigned once at creation time, acting as a static weight.
Relevance
cos(q, m)
Cosine similarity between the query embedding and the memory embedding. This is the standard semantic retrieval signal -- how topically related is this memory to the current situation?
score(m) = αrecency × recency(m) + αimportance × importance(m) + αrelevance × relevance(m)

The weighted sum of all three factors determines which memories are injected into the agent's prompt at decision time. This multi-factor approach prevents pure semantic similarity from dominating -- a highly relevant but ancient memory may be outranked by a moderately relevant but recent and important one. The system also supports reflection: periodically, the agent generates higher-level abstractions from clusters of recent memories, creating metacognitive entries that themselves enter the memory stream.

04 Post-Weng Extensions

Weng's taxonomy has not been updated since June 2023. Subsequent researchers have built upon, challenged, and extended her framework in significant ways. Each adds dimensionality that the original formulation lacked, particularly around memory operations (not just types), structural organization, and decision-theoretic grounding.

Author & Year Framework Key Contribution
Zhang et al. 2024 Survey of Memory in LLM-based Agents Introduces a three-axis taxonomy: sources (where memory comes from: internal knowledge, user interaction, external tools), forms (how memory is represented: natural language, embeddings, databases, structured knowledge graphs), and operations (what agents do with memory: read, write, reflect, consolidate, forget). The operations axis is the critical addition -- Weng focused on memory types but not the active processes that manage them.
Wu et al. 2025 3D-8Q Evaluation Framework Proposes three evaluation dimensions: knowledge (what does the agent remember?), time (how does recall degrade or improve over time?), and cognition (what cognitive processes govern memory use?). Within these, eight specific questions operationalize evaluation. First systematic attempt to benchmark agent memory quality rather than just retrieval accuracy.
Xu et al. 2025 A-Mem (Agentic Memory) Applies Zettelkasten principles (Luhmann's slip-box method) to agent memory. Each memory note is atomic, uniquely addressed, and linked to related notes. Memories self-organize through agent-driven indexing, tagging, and cross-referencing. Demonstrates that structured note-taking outperforms flat vector stores on complex reasoning tasks requiring multi-hop memory retrieval.
Du et al. 2026 POMDP Memory Framework Grounds agent memory in Partially Observable Markov Decision Processes. Memory is not just storage but a belief state that the agent maintains about its partially observable environment. Read/write/forget become actions in a decision-theoretic framework with explicit costs and rewards. Formalizes the intuition that remembering everything is not optimal -- the agent should actively choose what to forget based on expected future utility.

The trajectory from Weng (2023) through these extensions shows the field maturing from descriptive taxonomy (what types of memory exist) to operational frameworks (how memory should be managed) to formal decision theory (when memory operations are optimal). Each layer builds on Weng's foundation while addressing its limitations.

05 Key Facts

  • • Published: June 2023, on Lilian Weng's personal blog (lilianweng.github.io)
  • • Title: "LLM Powered Autonomous Agents" -- memory is one of three pillars alongside planning and tool use
  • • Updates: No revisions since original publication. The post remains a static snapshot of mid-2023 understanding
  • • Citation impact: The most-cited reference in the agent memory literature. Virtually every subsequent survey, framework, and system paper references the Weng taxonomy as baseline
  • • Affiliation at time: Head of Applied Research at OpenAI. The blog is personal, not an OpenAI publication
  • • Current role: Now at Thinking Machines (as of 2025). No longer at OpenAI
  • • Scope: The post covers the full agent stack, but the memory section became its most influential contribution. The planning section (Chain of Thought, Tree of Thoughts, LLM+P) and tool use section (MRKL, Toolformer, HuggingGPT) are also widely cited but have been more actively superseded by later work
  • • Limitations: No coverage of memory consolidation, forgetting, or operational lifecycle. Treats memory as static storage rather than active process. These gaps drove the post-Weng extensions above
