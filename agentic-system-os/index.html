02 — Privilege Separation

ADR-001: Graduated three-level architecture

Security scales with capability. Each level adds isolation without requiring the next. Level 1 ships with the scheduler, Level 2 with multi-agent routing, Level 3 when write actions span orgs.

Level 1
Single-Agent Validation
Phase 2d — Scheduler Foundation
Go-side validation layer before Claude CLI invocation. Recipient allowlists, injection regex on outputs, action type enum. The relay checks every outbound action against a static policy before execution.
~130 LOC · Zero extra cost · Fails closed
Level 2
Planning / Action Split
Phase 3a — Multi-Agent Routing
Planning agents are read-only: they analyze data and produce structured TaskRequest JSON. Action agents are write-capable but blind to raw source data. Per-tier tool whitelists enforce the boundary.
Per-tier whitelists · ~$0.05 per write-action
Level 3
Security Auditor
Phase 3b — Org Agent Watchers
A dedicated security agent intercepts TaskRequest JSONs between planning and action tiers. Compares raw inputs against proposed outputs. Builds self-improving threat intelligence from flagged patterns.

04 — Expert Panel Results

Six reviews shaped the architecture

Before implementation, the design was stress-tested by domain expert agents. Each reviewed the architecture from their specialty and produced concrete recommendations.

1
Agentic Systems Architect
Surveyed 7 frameworks and 4 academic papers. Pattern is validated: OpenClaw shows 323× attack resistance from isolation alone. No existing framework has built-in privilege separation at this level.
2
AI Security Architect
Air gap strength depends on handoff format: freeform = WEAK, structured JSON = STRONG. Designed 6-rule minimum viable security protocol. P0: structured JSON schema + per-tier tool whitelists.
3
Distributed Systems Engineer
Graduated 3-level path from single-agent validation to full tier separation. TaskRequest JSON schema with typed fields. Level 1 adds $0 cost; Level 2 adds ~$0.05 per write-action.
4
Adversarial Red Teamer
Produced 5 concrete attack scenarios. Level 1 → 2 is the biggest security jump (2 scenarios EXPOSED → BLOCKED). No scenario is ONLY caught by Level 3. Two attack classes require human judgment regardless.
5
Product Engineer
Designed a 4-tier notification model: Silent, Notify, Approve, Timed. Per-org autonomy dial. Quiet hours batching. Phase 2d ships Tier 1 only with an extensible foundation for escalation.
6
Adversarial Reviewer
Cross-validated all expert findings for internal consistency and blind spots. Corrected 1 finding, confirmed 54 coded observations. Quality gate: 3 of 4 divergence metrics passed.
