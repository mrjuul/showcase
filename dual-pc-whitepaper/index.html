One Brain, Two Machines

How to run Claude Code across two PCs with full memory continuity

scroll

// The Problem

Dev environments don't travel well

You have a laptop and a desktop. Same repos, same tools, same brain. But your AI coding assistant doesn't know what you did on the other machine. Skills, conventions, memory, MCP servers, secrets — all locked to one device.

Claude Code stores per-project memory using path-encoded directories. C:\Users\chris\ClaudeHub becomes a different slug than C:\Users\Juul\ClaudeHub. Different username = different memory = different brain.

// Architecture

Git-synced brain, per-machine body

Everything shareable lives in a single git repo (ClaudeHub). Machine-specific state stays local. Junctions and hardlinks make it seamless — Claude Code sees ~/.claude/skills/ but it's actually reading from ClaudeHub.

// Memory System

Claude Code's memory isn't one thing — it's four independent systems with different retrieval patterns. The architecture ensures 100% of critical context loads every session, with deeper history available on demand.

// The Bridge

How memories cross machines

SQLite can't be git-synced (binary, no merge). The solution: export observations to append-only JSONL, sync via vault, import on the other machine. Git handles concurrent appends cleanly — each line is independent.

Session lifecycle # Laptop session ends /reflect → exports 16 observations to vault/claude-mem/observations.jsonl /ship → commits + pushes ClaudeHub + vault (including JSONL) # Desktop session starts .bashrc → auto-pulls ClaudeHub + vault vault-pull.js → git pull + claude-mem-import.py claude-mem → reads local SQLite (now has laptop's observations) "Last session you completed Phase 1 of the dual-PC setup..."

Export (claude-mem-export.py)

Reads new observations from local SQLite. Appends to vault JSONL. Uses a watermark file to track what's already exported. Idempotent.

Import (claude-mem-import.py)

Reads JSONL from vault. Checks content_hash for deduplication. Inserts new records into local SQLite. Rebuilds FTS index.

// Results

What you get

Seamless switching

Walk from laptop to desktop. Open a terminal. Claude knows where you left off — SPEC.md changelogs + imported observations provide full context.

Zero manual sync

Auto-pull on terminal open. Auto-import on session start. The only explicit steps are /reflect and /ship — which you'd do anyway.

Clean git history

No more auto-save noise. Every vault commit is intentional, with a descriptive message. Session-end Stop hook retired.

Portable by design

Zero hardcoded paths. Junctions for dirs, hardlinks for files. Every reference uses ~, $HOME, or env vars. Works on any Windows username.
