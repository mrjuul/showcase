03 / Orchestration

Domain-Scoped Agents

5 agents, each scoped to a specific domain. Agents are invoked via the Agent tool with mode: dontAsk for batch operations or acceptEdits for read-only research.

ops Infrastructure health monitoring -- MCP servers, Docker containers, NAS services tech/ only
secretary Board meeting prep, agendas, financial summaries, minute drafting orgs/ only
config-auditor Config hygiene checks, CLAUDE.md size review, convention drift detection any domain
researcher Cross-org research via Google Workspace, Drive, Asana, email archives orgs/ only
reviewer Code and documentation review against CLAUDE.md conventions and standards tech/ only
04 / Boundaries

Hook Enforcement

PreToolUse hook enforces domain boundaries at runtime. All denials logged to hook-denials.jsonl.

142 lines bash
01 Email tool must match current working directory org domain
02 SSH blocked from orgs/ and personal/ workspaces
03 Google Workspace MCP blocked from tech/ workspace
04 Dangerous commands (sshpass, iptables, nmap, wsl) blocked from orgs/personal
05 Org-specific MCP servers blocked from personal/ workspace
06 Windows-MCP infrastructure tools blocked from orgs/ workspace
07 Hardcoded NAS credentials blocked everywhere -- must use env vars

06 / Persistence

Four-Layer Memory

Memory spans from ephemeral per-project facts to a permanent, searchable knowledge base. Each layer serves a different recall pattern.

1

Native Auto-Memory

Per-project facts, symlinked to vault. Claude writes automatically, loaded every session.

52 files
2

CLAUDE.md + SPEC.md

Conventions and project state. CLAUDE.md auto-loaded, SPEC.md read on entry. Human-curated.

28 + n files
3

Claude-Mem (SQLite)

Cross-session observations. Injected at SessionStart via hook. Searchable via MCP.

SQLite DB
4

Vault (Obsidian)

Permanent knowledge base. GitHub-synced, wiki-linked, accessed via Graphthulhu MCP.
